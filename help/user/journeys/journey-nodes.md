# 계정 여정 노드

[계정 여정을 만들고](journey-overview.md#create-an-account-journey) [대상자를 추가](journey-overview.md#add-the-account-audience-for-your-journey)한 후 노드를 사용하여 여정을 빌드하십시오. 여정 맵은 다중 단계 B2B 마케팅 사용 사례를 빌드할 수 있는 캔버스를 제공합니다.

다양한 작업, 이벤트 및 오케스트레이션 노드를 다중 단계 크로스 채널 시나리오로 결합하여 계정 여정을 구축합니다. 여정의 각 노드는 논리적 경로를 따라 단계를 나타냅니다.

## 계정 대상자 노드

[계정 대상](journey-overview.md#add-the-account-audience-for-your-journey) 노드는 여정에 대한 입력 계정 대상(Adobe Experience Platform에서 만들고 관리됨)을 정의합니다. 이 노드는 항상 첫 번째 노드이며 기본적으로 자동으로 만들어집니다.

## 조치 취하기

이메일 보내기, 점수 변경 등과 같은 작업을 실행합니다.

**계정에 대한 작업**: 이 작업은 이 경로에 있는 계정의 일부인 모든 사용자에게 적용됩니다.

**사용자에 대한 작업**: 이 작업은 이 경로의 모든 사용자에게 적용됩니다. 사람에 대한 작업은 사람에 의한 분할 경로 또는 계정에 의한 분할 경로 내에서 사용할 수 있습니다.

| 노드 컨텍스트 | 함수 | 제한 |
| ------------ | -------- | ----------- |
| [사람](#add-a-people-action) | 구매 그룹에 할당 | 솔루션 관심 항목 선택<br/>역할 선택 |
| | 구매 그룹에서 제거 | 솔루션 관심 항목 선택 |
| | SMS 보내기 | SMS 만들기 |
| | Marketo Engage 요청 캠페인에 추가 | Marketo Engage 작업 영역 선택<br/>요청 캠페인 선택 |
| | Marketo Engage에서 사람 파티션 변경 | 새 파티션 |
| | 사람 즐거운 순간 | 유형<br/>설명 |
| | 점수 변경 | 점수 이름<br/>변경 |
| | 이메일 보내기 | 새 전자 메일 만들기<br/>Marketo Engage에서 전자 메일 선택 |
| [계정](#add-an-account-action) | 판매 알림 보내기 | 솔루션 관심 항목 선택<br/>전자 메일 보내기 |
| | (기타) 여정에 계정 추가 | 라이브 계정 여정 선택 |
| | 구매 그룹 상태 업데이트 | 솔루션 관심 분야<br/>상태(필수, 최대 50자) |
| | (현재) 여정에서 계정 제거 | 라이브 계정 여정 선택 |
| | 계정 관심 순간 | 유형(전자 메일, 마일스톤 또는 웹)<br/>설명(선택 사항) |
| | 계정 변경 데이터 값 | 특성 <br/>새 값 선택 |

### 계정 작업 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 작업 수행]**&#x200B;을 선택합니다.

   ![여정 노드 추가 - 경로 분할](./assets/add-node-action.png){width="400"}

1. 오른쪽의 노드 속성에서 작업에 대해 **[!UICONTROL 계정]**&#x200B;을(를) 선택합니다.

1. 목록에서 작업을 선택하고 작업에 대한 값을 설정합니다.

   ![여정 노드 - 계정에서 작업 수행](./assets/node-take-action-account.png){width="700" zoomable="yes"}

### 사람 작업 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 작업 수행]**&#x200B;을 선택합니다.

1. 오른쪽의 노드 속성에서 작업에 대해 **[!UICONTROL 사람]**&#x200B;을(를) 선택합니다.

1. 목록에서 작업을 선택하고 작업에 대한 값을 설정합니다.

![여정 노드 - 사용자에 대한 작업 수행](./assets/node-take-action-people.png){width="700" zoomable="yes"}

## 이벤트 수신

이벤트가 발생할 때 대상을 여정의 다음 단계로 이동합니다.

* 여정이 이 이벤트를 기다리는 시간을 정의할 수도 있습니다. 시간 제한 후 여정이 종료됩니다.
* 또한 시간 초과 경로에 다른 노드를 추가하도록 선택할 수 있습니다.

**계정에서 이벤트 수신**: 계정에서 한 명 이상의 사용자가 이벤트를 트리거하면 여정에서 다음 단계로 이동합니다.

**피플에서 이벤트를 수신**: 피플에 대한 이벤트는 계정 경로에서만 적용할 수 있으며, 피플 노드별 분할에는 사용할 수 없습니다.

| 노드 컨텍스트 | 함수 | 제한 |
| ------------ | -------- | ----------- |
| [사람](#add-a-people-event) | 데이터 값 변경 | 특성<br/>추가 제약 조건(선택 사항)<br/>시간 초과(선택 사항) |
| | 이메일의 링크 클릭 | 이메일<br/>추가 제약 조건(선택 사항)<br/>시간 초과(선택 사항) |
| | 구매 그룹에 할당됨 | 솔루션 관심<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 이메일 열기 | 이메일<br/>추가 제약 조건(선택 사항)<br/>시간 초과(선택 사항) |
| | 점수가 변경됨 | 점수 이름<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 구매 그룹에서 제거됨 | 솔루션 관심 분야<br/>활동 날짜(선택 사항)<br/>시간 초과(선택 사항) |
| [계정](#add-an-account-event) | 구매 그룹 상태 변경 | 솔루션 관심<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 완성도 점수 변경 | 솔루션 관심<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 계정에는 흥미로운 순간이 있었습니다. | Type<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 참여 점수 변경 | 솔루션 관심<br/>추가 제약 조건(선택 사항)<br/>시간 제한(선택 사항) |
| | 계정 데이터 값 변경 | 특성<br/>추가 제약 조건(선택 사항)<br/>시간 초과(선택 사항) |

### 계정 이벤트 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 이벤트 수신]**&#x200B;을 선택합니다.

1. 오른쪽의 노드 속성에서 이벤트 유형으로 **[!UICONTROL 계정]**&#x200B;을(를) 선택합니다.

   ![여정 노드 - 계정에서 이벤트를 수신](./assets/node-listen-events-account.png){width="700" zoomable="yes"}

1. 목록에서 이벤트를 선택합니다.

1. **[!UICONTROL 이벤트 편집]**&#x200B;을 클릭하고 이벤트에 대한 세부 정보를 정의하십시오.

### 인물 이벤트 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 이벤트 수신]**&#x200B;을 선택합니다.

1. 오른쪽의 노드 속성에서 이벤트 유형으로 **[!UICONTROL 사람]**&#x200B;을(를) 선택합니다.

   ![여정 노드 - 사용자의 이벤트를 수신](./assets/node-listen-events-people.png){width="700" zoomable="yes"}

1. 목록에서 이벤트를 선택합니다.

1. **[!UICONTROL 이벤트 편집]**&#x200B;을 클릭하고 이벤트에 대한 세부 정보를 정의하십시오.

### 이벤트 노드에 시간 초과 추가

필요한 경우 여정이 이벤트를 기다리는 시간을 정의합니다. 시간 제한 후 여정이 종료됩니다.

1. 시간 초과 토글을 활성화합니다.

1. 이벤트가 시간 초과되기 전에 발생할 때까지 여정이 기다리는 기간을 선택합니다.

   여기서 경로를 종료하거나 다른 경로를 설정하여 다른 동작을 수행하도록 선택할 수 있습니다.

1. 이벤트가 발생하지 않을 때 여정에 적용할 수 있는 작업 및 이벤트를 추가할 수 있는 새 경로를 만들려면 **[!UICONTROL 시간 제한 경로 설정]** 확인란을 선택하십시오.

   ![여정 이벤트 노드 - 시간 제한 경로 설정](./assets/node-event-timeout-set-path.png){width="700" zoomable="yes"}

## 경로 분할

필터 조건에 따라 대상자를 분할합니다.

>[!NOTE]
>
>최대 25개의 경로가 지원됩니다.

**계정별로 경로 분할**: 계정별로 분할된 경로에는 계정과 사용자 작업 및 이벤트가 모두 포함될 수 있으며 이러한 경로는 추가로 분할할 수 있습니다.

_계정 노드별 분할 경로는 어떻게 작동합니까?_

* 분할 경로 노드를 추가하고 _계정_&#x200B;을(를) 선택하면 추가된 각 경로에는 각 에지에 노드를 추가할 수 있는 최종 노드가 포함됩니다.
* 중첩된 방식으로 경로를 계정별로 반복적으로 분할할 수 있습니다. 분할 경로에는 기본 경로를 추가하지 않는 옵션이 포함되어 있습니다.
* 분할된 경로 중 하나에 적합하지 않은 계정/사람은 여정에서 앞으로 이동하지 않습니다.
* 이러한 경로는 병합 노드를 사용하여 결합할 수 있습니다.

![여정 노드 - 계정별로 경로 분할](./assets/node-split-paths-account.png){width="700" zoomable="yes"}

**사람별로 경로 분할**: 사람이 분할한 경로에는 사람 작업만 포함될 수 있으며 이 경로는 다시 분할할 수 없습니다. 경로는 자동으로 다시 연결됩니다.

_people 노드별 분할 경로는 어떻게 작동합니까?_

* 사람 노드로 경로 분할은 그룹화된 노드입니다. 대상자에 있는 모든 사람이 자신이 속한 계정의 컨텍스트를 손실하지 않고 다음 단계로 이동할 수 있도록 자동으로 병합됩니다.
* 사람에 대한 분할 경로는 중첩될 수 없으며, 이렇게 그룹화된 노드에 있는 경로에 사람에 대한 분할 경로를 추가할 수 없습니다.
* 분할 경로에는 기본 경로를 추가하지 않는 옵션이 포함되어 있습니다. 자격이 없는 계정/사람은 여정에서 앞으로 이동하지 않습니다.

![여정 노드 - 사람별로 경로 분할](./assets/node-split-paths-people.png){width="700" zoomable="yes"}

| 노드 컨텍스트 | 경로 조건 | 설명 |
| ------------ | -------- | ----------- |
| [사람](#add-a-split-path-by-people-node) | 개인 속성 | |
| | 변경된 데이터 값(예: 활동 기록 필터) | |
| | 이메일 열림 | |
| | 이메일의 클릭한 링크 | |
| | 웹 페이지에서 클릭한 링크 | |
| | 즐거운 시간 보냈어요 | |
| | 구매 그룹 구성원 | |
| [계정](#add-a-split-path-by-account-node) | 계정 데이터 값 변경(예: 활동 내역 필터링) | |

### 계정 노드별 분할 경로 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 경로 분할]**&#x200B;을 선택합니다.

   ![여정 노드 추가 - 경로 분할](./assets/add-node-split.png){width="300"}

1. 오른쪽의 노드 속성에서 분할을 위해 **[!UICONTROL 계정]**&#x200B;을(를) 선택합니다.

1. _[!UICONTROL 경로 1]_&#x200B;에 적용할 수 있는 조건을 정의하려면 **[!UICONTROL 조건 적용]**&#x200B;을 클릭하십시오.

   ![경로 노드 분할 - 조건 추가](./assets/node-split-properties-apply-condition.png){width="500"}

1. 조건 편집기에서 하나 이상의 필터를 추가하여 분할 경로를 정의합니다.

   * 왼쪽 탐색에서 필터 속성을 끌어서 놓고 일치 정의를 완료합니다.

   * 맨 위에 있는 **[!UICONTROL 필터 논리]**&#x200B;를 적용하여 조건을 미세 조정하십시오. 모든 속성 조건 또는 임의의 조건을 일치시키도록 선택합니다.

     ![경로 노드 분할 - 조건 필터 논리](./assets/node-split-conditions.png){width="700" zoomable="yes"}

   * **[!UICONTROL 완료]**&#x200B;를 클릭합니다.

1. 경로를 더 추가하려면 **[!UICONTROL 경로 추가]**&#x200B;를 클릭하고 이전 단계를 반복하여 이 경로에 적용할 수 있는 조건을 추가합니다.

   이러한 조건을 기반으로 각 경로에 레이블을 지정하거나 기본 레이블을 사용할 수도 있습니다.

1. (선택 사항) 다른 경로에 대해 적합하지 않은 계정의 기본 경로를 추가합니다. 그렇지 않으면 이러한 계정에 대한 여정이 종료됩니다.

   ![분할 경로 노드 속성 - 기타 계정](./assets/node-split-properties-other-accounts.png){width="700" zoomable="yes"}

### 사람 노드별 분할 경로 추가

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 경로 분할]**&#x200B;을 선택합니다.

   ![여정 노드 추가 - 경로 분할](./assets/add-node-split.png){width="300"}

1. 오른쪽의 노드 속성에서 분할을 위해 **[!UICONTROL 사람]**&#x200B;을(를) 선택합니다.

1. _[!UICONTROL 경로 1]_&#x200B;에 적용할 수 있는 조건을 정의하려면 **[!UICONTROL 조건 적용]**&#x200B;을 클릭하십시오.

1. 조건 편집기에서 하나 이상의 필터를 추가하여 분할 경로를 정의합니다.

   * 왼쪽 탐색에서 필터 속성을 끌어서 놓고 일치 정의를 완료합니다.

   * 맨 위에 있는 **[!UICONTROL 필터 논리]**&#x200B;를 적용하여 조건을 미세 조정하십시오. 모든 속성 조건 또는 임의의 조건을 일치시키도록 선택합니다.

   * **[!UICONTROL 완료]**&#x200B;를 클릭합니다.

1. 경로를 더 추가하려면 **[!UICONTROL 경로 추가]**&#x200B;를 클릭하고 이전 단계를 반복하여 이 경로에 적용할 수 있는 조건을 추가합니다.

   이러한 조건을 기반으로 각 경로에 레이블을 지정하거나 기본 레이블을 사용할 수도 있습니다.

1. 마지막으로 위의 경로에 적합하지 않은 사람에 대한 기본 경로를 추가할 수 있습니다. 그렇지 않은 경우 이러한 사용자에 대한 여정이 종료됩니다

사용자 수준에서 대상을 분할하는 각 경로에 대해 조건이 정의된 경우 사용자에 대해 수행할 작업을 추가할 수 있습니다.

>[!NOTE]
>
>대상자를 사람별로 분할하면 사람 작업만 추가할 수 있습니다.

## 대기

다음 단계로 이동하기 전에 특정 기간 동안 기다립니다.

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 대기]**&#x200B;를 선택합니다.

1. 오른쪽의 노드 속성에서 여정이 경로의 다음 노드로 진행되기 전에 대기할 시간을 **[!UICONTROL Duration]**&#x200B;으로 설정합니다.

![여정 노드 - 대기](./assets/node-wait.png){width="700" zoomable="yes"}

## 경로 병합

이 노드를 사용하여 여정의 다른 경로를 병합하고 병합을 취소할 수 있습니다.

1. 여정 편집기로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 경로 분할]**&#x200B;을 선택합니다.

1. 분할 노드를 클릭하여 오른쪽에서 해당 속성을 엽니다.

1. 세 개의 경로를 만들려면 [!UICONTROL 경로 추가]를 클릭하십시오.

1. 각 경로에 작업과 이벤트의 조합을 추가합니다.

1. 이러한 경로 중 하나에 대한 더하기( **+**) 아이콘을 클릭하고 표시된 옵션에서 **[!UICONTROL 병합]**&#x200B;을(를) 선택합니다.

   ![여정 노드 - 병합 경로](./assets/node-plus-icon-merge-paths.png){width="400"}

1. 병합 노드 속성에서 병합할 경로를 선택합니다.

   ![여정 노드 - 병합 경로](./assets/node-merge-select-paths.png){width="600" zoomable="yes"}

   이제 선택한 경로의 계정이 단일 경로로 결합되어 여정을 계속 진행할 수 있도록 경로가 병합된다는 것을 확인해야 합니다.

1. 필요한 경우 병합 노드 속성으로 다시 이동하고 제거할 경로에 대한 확인란을 선택 취소하여 경로 병합을 취소할 수 있습니다.
