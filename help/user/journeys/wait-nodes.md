# 대기 노드

다음 단계로 이동하기 전에 특정 기간 동안 여정 진행을 일시 중지하려면 _대기_ 노드를 사용하십시오.

대기 시간을 정의하는 방법에는 두 가지가 있습니다.

* 여정의 다음 노드로 이동하려는 특정 날짜
* 상대적 기간(분, 시간, 일, 주 또는 개월 수)

## 대기 노드 추가

1. 여정 맵으로 이동합니다.

1. 경로에서 더하기(**+**) 아이콘을 클릭하고 **[!UICONTROL 대기]**&#x200B;를 선택합니다.

   ![여정 노드 추가 - 대기](./assets/add-node-wait.png){width="440"}

1. 오른쪽의 노드 속성에서 여정이 경로의 다음 노드로 진행되기 전에 대기할 시간을 **[!UICONTROL Type]**&#x200B;로 설정합니다.

   * **[!UICONTROL 기간]** - 대기 노드의 시작과 종료 사이에 경과할 특정 일, 시간 또는 분을 정의합니다.
   * **[!UICONTROL 날짜]** - 종료에 대한 특정 날짜 및 시간을 지정합니다.

   ![여정 노드 - 대기](./assets/node-wait.png){width="500"}

## 고급 대기 설정

**[!UICONTROL 다음에서 끝나야 함]** 옵션을 사용하여 _고급 대기 단계_&#x200B;를 구성하고 메시지가 최적의 순간에 사람 및 계정 구성원에게 도달하는지 확인하십시오. 이 구성을 사용하면 개인 또는 계정이 대기 단계를 종료하고 여정의 다음 노드로 진행되는 시기를 정확하게 제어할 수 있습니다. 시작부터 종료까지 고정된 시간 또는 일 수가 아니라 특정 시간 및 특정 요일에 실행되도록 작업을 예약할 수 있습니다.

_고급 대기 단계_&#x200B;를 사용하여 대기 시간이 아니라 사용자 또는 계정이 종료되는 **_시기_**&#x200B;를 정의합니다.

![여정 노드 - 고급 대기 단계](./assets/node-wait-advanced.png){width="500"}

>[!AVAILABILITY]
>
>고급 대기 설정은 [!DNL Journey Optimizer B2B Edition]간소화된 아키텍처[에서 프로비저닝된 &#x200B;](../simplified-architecture.md) 환경에서 사용할 수 있습니다.

### 대기 유형

| 대기 유형 | 설명 | 구성 |
| --------- | ----------- | ------------- |
| **특정 시간** | 특정 시간(예: 오전 9:00)까지 보류 | 시간(시간 및 분)을 설정합니다. 해당 시간대의 다음 발생 시(선택한 시간대에 대해) 종료합니다. |
| **특정 요일** | 특정 날짜(예: 화요일)까지 보류 | 요일을 선택합니다. 시간을 지정하지 않으면 선택한 시간대의 자정에 일치하는 다음 날에 종료됩니다. |
| **일 범위 또는 조합** | 범위 내(예: 월요일-금요일) 또는 지정된 날짜 중 하나까지 보류 | 목표 일자를 선택합니다. 시간을 지정하지 않으면 선택한 시간대의 자정에 일치하는 다음 날에 종료됩니다. |
| **시간 + 일 조합** | 정확한 예약을 위해 두 옵션을 결합합니다(예: 화요일 오전 10:00). | 목표 일수를 선택하고 목표 시간을 설정합니다. 다음 날/시간 발생 시 종료(선택한 시간대에 대해). |

### 일반적인 시나리오

다음 시나리오는 일반적인 시나리오를 대기 노드 구성에 적용하는 방법을 보여 줍니다.

+++업무 시간 중 이메일 도착

**시나리오:** 직장에서 전자 메일을 읽는 B2B 고객에게 마케팅합니다. 모든 이메일이 업무 시간 중에 도착하기를 원합니다.

**해결 방법:** 평일(월~금요일) 오전 9시:00에 리드를 릴리스하도록 대기 단계를 구성하십시오. 잠재 고객이 대기 노드에 들어와도 업무 시간 중에 이메일을 받게 됩니다.

+++

+++동적 대상에 대한 일관된 전송 시간

**시나리오:** 새 계정 또는 잠재 고객 자격으로 대상이 매일 변경됩니다. 자격 조건을 충족한 시기에 관계없이 모든 잠재 고객이 동시에 첫 번째 이메일을 받도록 할 수 있습니다.

**해결 방법:** 대기 단계를 특정 시간(예: 오전 10:00)에 종료되도록 설정합니다. 자정이든 정오든 모든 리드는 오전 10:00에 대기 단계를 함께 종료합니다.

+++

+++SLA 호환 후속 작업

**시나리오:** 영업팀에서 마케팅 자격을 갖춘 계정 리드에 대한 후속 작업을 수행할 수 있는 2영업일 SLA을 제공합니다. 주말은 포함되지 않습니다.

**해결 방법:** 영업일에만 리드를 릴리스하도록 대기 단계를 구성합니다. 금요일 적격 리드는 주말이 아닌 월요일이나 화요일에 후속 조치를 위해 라우팅됩니다.

+++

### 시작 및 종료 예

| 대기 구성 | 계정/잠재 고객 입력 | 계정/리드 종료 |
| ------------------ | ------------------- | ------------------ |
| 매일 오전 9:00 | 월요일 오전 11:00 | 화요일 오전 9:00 |
| 매일 오전 9:00 | 월요일 오전 7:00 | 월요일 오전 9:00 |
| 화요일, 정해진 시간 없음 | 금요일 오후 3:00 | 화요일 오전 12:00 |
| 월요일-금요일 오전 10:00 | 토요일 오후 2:00 | 월요일 오전 10:00 |
| 월요일-금요일 오전 10:00 | 수요일 오전 8:00 | 수요일 오전 10:00 |
