# 간단한 방법으로 프로그램 만들기

[!DNL Adobe Journey Optimizer B2B Prime]에서 [_program_](../marketing/programs.md)&#x200B;은(는) 캠페인 내의 여정, 사용자 목록, 토큰 및 구성에 대한 최상위 컨테이너입니다. [채팅 인터페이스](./chat-interface.md)에서 프로그램 제작 기술은 프로그램 자체, 하위 폴더, 토큰, 사람 목록, 사람 여정 등 간단한 대화에서 전체적인 구조를 만듭니다.

* **스킬** - `program-creation`
* **호출** - 요약을 업로드하고 _이 요령에서 프로그램 만들기_ / _요령에서 프로그램 만들기_&#x200B;를 입력하거나 캠페인을 직접 설명하십시오.
* **읽기/쓰기 대상** - [!DNL Journey Optimizer B2B Prime]; 테넌트에서 프로그램 유형 구성도 읽기

## 간략한 업로드

채팅 입력에서 _첨부_ 아이콘을 클릭하여 파일을 업로드한 다음 원하는 내용을 설명하십시오. 브리프의 텍스트가 추출되고 컨텍스트로서 도우미에게 전달됩니다(채팅에서 &quot;캠페인 브리프 업로드됨: 파일 이름(NkB)&quot; 표시기가 표시됨).

### 지원되는 파일 유형

| 형식 | 비헤이비어 |
|---|---|
| `.txt`, `.md` | 브라우저에서 직접 읽기 |
| `.pdf`, `.docx`, `.xlsx`, `.json`, `.csv` | 백엔드 텍스트 추출 서비스로 전송됨 |

이름이 *brief*&#x200B;인 파일은 캠페인 개요로 자동 태그가 지정됩니다. 그렇지 않으면 도우미가 해당 파일을 컨텍스트에서 유추합니다.

### 크기 제한

간단한 텍스트는 도우미에 도달하기 전에 약 30,000자(~7.5K 토큰)로 잘립니다. 매우 긴 준비는 해당 시점에서 잘립니다(`(truncated to 29KB)` 메모로 표시됨). 필수 캠페인 세부 정보를 앞에 추가합니다.

## 권장 간략한 내용

흐름 접수 는 다음 입력에 대한 브리프를 읽습니다. 가능한 한 많이 포함합니다.

* 프로그램 이름 및 간단한 설명/목적
* 프로그램 유형 신호(예: 박람회, 이벤트, 웨비나, 로드쇼, 회의 또는 육성)
* 인물 목록에 대한 대상 기준
* 여정 디자인 — 카운트, 시작 기준, 접점/시간 및 Go-Live 날짜
* 토큰(이벤트 날짜, 장소, 제목 줄 등 재사용 가능한 값)

AI 어시스턴트는 빌드하기 전에 누락된 모든 것을 요청합니다.

## 빌드 단계

이 스킬은 **INTAKE → APPROVE → BUILD**&#x200B;의 세 단계로 실행됩니다. 승인할 때까지 아무 것도 만들어지지 않습니다.

### Intake

도우미는 개요에서 다음 내용을 추출하고 누락된 내용이 있는지 묻습니다.

* 프로그램 이름
* 상위 폴더(지정되지 않은 경우 작업 영역 루트로 기본값)
* 설명
* 프로그램 유형(가능한 경우 간단한 문구에서 유추)
* 생성할 토큰(이름, 유형, 값)
* 사용자 목록 대상 기준
* 여정 — 수, 항목 기준, 터치포인트, Go-Live 날짜

### 승인

진행하기 전에 도우미가 확인 요약을 제공합니다.

`I'll create {program} in folder {id}, with {N} token(s), {N} dynamic people list(s), and {N} journey(s). Shall I proceed?`

명확한 진행으로 응답할 때까지 생성 도구가 호출되지 않습니다(예: _진행_, _진행_, _빌드_, _승인_ 및 _예_).

### 빌드

빌드 단계는 고정된 순서로 실행됩니다. 각각은 이전 단계에서 생성된 ID에 따라 다릅니다.

| 단계 | 설명 | 도구 | 출력 |
|---|---|---|---|
| **1. 폴더** | 이름으로 상위 폴더를 확인하거나 작업 영역 루트를 사용합니다. 새 하위 폴더를 만들 수 있습니다. | `getRootTree`, `browseFolders`, `createFolder` | 폴더 이름 + ID |
| **1.5. 프로그램 유형** | 테넌트 프로그램 유형을 조회하고 개요 일치 항목을 선택합니다. | `browseProgramTypes` | 형식 이름 + `programTypeId` |
| **2. 프로그램** | 선택한 형식에 바인딩된 확인된 폴더 아래에 프로그램을 만듭니다. | `createProgram` | 프로그램 이름, ID, 유형 |
| **3. 토큰** | 프로그램에 각 `my.*` 토큰을 만듭니다. | `createProgramToken` | 토큰 이름 |
| **4. 사용자 목록** | 대상 기준에서 속성 기반 규칙을 생성하고 동적 사람 목록을 만듭니다. | `generate_ruleset`, `createSmartListWithRules` | 목록 이름 + `smartListId` |
| **5. 여정** | 4단계 목록으로 연결된 개인 대상자 입력 노드를 사용하여 각 개인 여정을 만듭니다. | `create_journey` | 여정 이름, ID |
| **6. 게시** | 라이브 날짜가 설정되고 확인된 경우 여정을 게시하거나 예약합니다. | `publish_journey_with_dates` | 라이브/예약된 여정 |
| **7. QA** | 선택적 유효성 검사 미리 보기 및 전체 QA 검사 | `qa_program_preview`, `qa_program` | 합격/불합격/경고 보고서 |

**_단계 종속성:_**

* 토큰, 목록 또는 여정을 첨부하려면 프로그램 ID(2단계)가 필요합니다.
* 4단계의 `smartListId`이(가) 5단계의 `create_journey`(으)로 전달됩니다. 생략하면 여정의 대상 항목 노드가 비어 있습니다.
* 여정 프롬프트는 항상 _Person 대상 노드를 진입점으로 시작_&#x200B;하여 채워진 시작 노드를 확인합니다.

## 출력 에셋

### 프로그램

최상위 컨테이너입니다. 세부 사항 보기는 다음 탭을 노출합니다.

| 탭 | 콘텐츠 |
|---|---|
| **개요** | 이름, 설명, 프로그램 유형, 상태, 폴더 위치, 타임스탬프 |
| **특성** | 유형 정의 사용자 정의 필드(프로그램 유형이 활성화하는 경우에만) |
| **토큰** | 이 프로그램의 토큰 범위가 `my.*`개 |
| **여정** | 프로그램에 포함된 여정 |

### 토큰

재사용 가능한 `my.*`개 값(예: `my.eventDate`). 각 유형에는 `text`, `date`, `rich text`, `score` 또는 `number` 중 하나와 값이 있습니다. 이메일과 콘텐츠 내에서 프로그램의 범위 내에서 해결됩니다.

### 사용자 목록

프로그램에서 생성된 속성 기반 규칙을 통해 대상 기준에서 빌드된 동적(스마트) 사람 목록입니다.

### 여정

개인 여정으로 시작하여 4단계 사용자 목록에 연결된 사용자 대상 항목 노드를 시작으로 개요의 터치포인트가 이어집니다(예: _1일 후 환영 이메일 보내기, 3일 후 후속 작업_). Go-Live 날짜가 설정되면 여정을 즉시 게시하거나 예약할 수 있습니다.

## 프로그램 유형 확인

프로그램 생성 시 테넌트 정의 프로그램 유형에 **영구적으로 바인딩**&#x200B;됩니다. 나중에 변경할 수 없습니다. 흐름은 모든 경우에 `browseProgramTypes`을(를) 통해 형식을 명시적으로 확인합니다.

| 사례 | 비헤이비어 |
|---|---|
| **여러 유형을 사용할 수 있음** | 간략한 단어를 유형에 일치시킵니다(예: tradeshow/booth/expo = *Tradeshow*/*Event*; 웨비나 = *Webinar*/*Event*; 육성/드립 = *Groose*; 명확한 신호 없음 = *Default*). 일치하는 항목이 없으면 도우미가 사용 가능한 유형을 나열하고 묻습니다. |
| **기본 전용 테넌트** | *기본값*&#x200B;을 사용하고 관리자가 사용자 지정 [프로그램 유형](../admin/program-types.md)을 추가할 수 있음을 알려줍니다. |
| **구성된 형식 없음** | 중지 — 생성에 실패합니다. 재시도하기 전에 프로그램 유형을 프로비저닝하라는 메시지를 관리자에게 보냅니다. |

## 기본값

| 설정 | 기본 |
|---|---|
| **폴더** | Workspace 루트(폴더가 지정되지 않은 경우) |
| **프로그램 유형** | *기본*(짧은 신호 및 수동 선택이 없는 경우) |
| **Personalization 특성** | 파생 사용자, 파생 의도, 기본적으로 포함된 참여 계층(옵트아웃 사용 가능) |
| **게시** | 향후 Go-Live 날짜에 대해 자동으로 사용되지 않음 - 수동 `ACTIVATE journey by {date}` 단계로 기록됨. 지금 시작 및 확인된 경우에만 즉시 게시 |
| **승인 게이트** | 명시적 승인 전에 생성된 에셋 없음 |

## 제한 사항

| 제한 사항 | 세부 정보 |
|---|---|
| **짧은 길이 상한** | ~30K 자, 잘린 팬티가 후행 콘텐츠를 잃음 — 필수 요소를 먼저 넣기 |
| **변경할 수 없는 프로그램 유형** | 생성 후에는 변경할 수 없습니다. 승인하기 전에 올바른 유형을 확인하십시오. |
| **필수 내부 필드** | `parent` 및 `programTypeId`은(는) 항상 설정되어 있습니다. `parent`을(를) 생략하면 액세스가 거부됩니다. 생략 유형은 자동으로 *기본값*(으)로 돌아갑니다. |
| **여정에 사용자 목록 필요** | 4단계의 `smartListId`은(는) 5단계에서 필수입니다. 흐름은 이를 적용합니다. |
| **비동기 목록 구성원** | 정적(기준에서) 목록이 몇 분 동안 채워집니다(인스턴트 아님). |
| **오류 처리** | 도구 실패가 정확한 오류와 함께 보고됩니다. 도우미는 입력을 확인하고 다시 시도하라는 메시지를 표시합니다. |
| **이름 충돌** | 자동으로 해결되지 않음 - 다른 이름이 요청됩니다. |
| **범위** | Journey Optimizer B2B Prime 전용. Marketo 사용자는 별도의 프로그램 제작/프로그램 계획 기술을 사용해야 합니다 |


## QA 검사

빌드가 완료되면 AI 어시스턴트는 다음을 제공합니다.

> &quot;실행 전에 QA 검사를 실행하여 모든 것을 확인하시겠습니까?&quot;

`qa_program_preview` 실행 후 `qa_program`을(를) 확인합니다. 이 프로세스는 권장 다음 단계와 함께 검사 통과, 실패 및 모든 경고 보고서를 반환합니다.
